# Unit 17 MSW Browser Worker Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Prepare browser MSW usage by generating the service worker file and adding a dev-only opt-in worker bootstrap.

**Architecture:** Keep MSW setup in `shared/api/mocks`. The app entrypoint may call a small bootstrap helper, but MSW must never start in production and must not replace the existing Vitest node server setup.

**Tech Stack:** Vite, React 19, TypeScript, MSW 2, Vitest.

### Task 2: Add Browser Worker Bootstrap Helper

**Files:**
- Create or modify: `src/shared/api/mocks/startWorker.ts`

- [ ] **Step 1: Implement opt-in worker start**

Use this behavior:

```ts
const shouldEnableMsw = import.meta.env.DEV && import.meta.env.VITE_ENABLE_MSW === 'true';

export const startMockWorker = async (): Promise<void> => {
  if (!shouldEnableMsw) return;
  const { worker } = await import('./browser');
  await worker.start({ onUnhandledRequest: 'bypass' });
};
```

- [ ] **Step 2: Avoid production start**

Confirm the function returns immediately when:

- `import.meta.env.DEV` is false
- `VITE_ENABLE_MSW` is not `'true'`

- [ ] **Step 3: Verify typecheck**

Run:

```bash
pnpm typecheck
```

Expected: PASS.

### Task 3: Wire App Bootstrap

**Files:**
- Modify: `src/main.tsx`

- [ ] **Step 1: Call helper before render**

Recommended structure:

```ts
import { startMockWorker } from '@shared/api/mocks/startWorker';

void startMockWorker().finally(() => {
  createRoot(document.getElementById('root')!).render(
    <StrictMode>
      <App />
    </StrictMode>,
  );
});
```

If public API rules make this import invalid, expose the helper through an appropriate public API or use a local app bootstrap helper that still respects FSD direction.
